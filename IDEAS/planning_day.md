# IDEAS — Планирование дня (`planning_day.md`)

**Status**

- Уже реализовано:
  - вечернее и утреннее планирование через `day_plans` / `day_plan_items` и модуль `handlers/day_plan.py`;
  - интеграция планов в `/today` (важные / дополнительные дела, синхронизация с очками);
  - терминальный симулятор сценариев планирования.
- Кандидаты на ближайшие спринты:
  - мягкое онбординг‑пояснение, как пользоваться планированием;
  - расширение связки «план дня → рутины / дела по дому»;
  - более гибкие категории дел (учёба, работа, быт, отдых).

Документ нужен, чтобы не потерять идеи про планирование вне конкретных спринтов.

---

## 0. Роль HiDL в планировании дня

HiDL выступает как мягкий напарник:

- помогает не забыть 3–5 ключевых дел;
- не превращает жизнь в бесконечный список задач;
- уважает усталость и щадящий режим.

Принципы:

- максимум 3 важных дела + несколько дополнительных;
- возможность «свернуть» план, если сил мало;
- связка с очками — не как оценка человека, а как видимый прогресс.

## 1. Вечернее планирование

Идея:

- вечером коротко набрасываем завтра:
  - 1–3 важных дела;
  - несколько «если будут силы».
- утром бот напоминает план и предлагает подкорректировать.

Реализация сейчас:

- таблицы `day_plans` и `day_plan_items`;
- модуль `handlers/day_plan.py`:
  - вечерний сценарий;
  - утренний пинг в планировщике.

Next sprint candidates:

- [ ] Добавить явные категории (работа/учёба/дом/здоровье/отдых);
- [ ] Дать возможность быстро переносить невыполненные дела на следующий день;
- [ ] Тонко согласовать тексты с щадящим режимом (если человек на паузе — план минимальный).

## 2. Связь плана дня с рутинами и домом

Идея:

- план дня не отдельно от рутин:
  - утренняя рутина может включать «посмотреть план»;
  - дела по дому могут подтягиваться в план как «фоновые» задачки;
- бот помогает не тащить всё в голове.

Текущий статус:

- `/today` уже показывает прогресс по плану (важные/дополнительные);
- из модуля `home_tasks` можно подтягивать регулярку в общий обзор.

Next sprint candidates:

- [ ] Кнопки «отправить в план дня» из разных разделов (дом, финансы, еда);
- [ ] режим «минимальный план» при щадящем режиме;
- [ ] небольшая статистика по выполнению планов (в модуле `stats`).
